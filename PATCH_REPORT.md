# THE LAST GUILD v0.6.1 — STABILITY & PERFORMANCE

## Цель

Остановить бесконтрольное разрастание систем, стабилизировать долгую симуляцию, ускорить загрузку и дать игроку полный контроль над тяжёлыми механиками мира.

## Реализовано

### Code splitting

- React-экраны и модальные окна переведены на `lazy()` и `Suspense`;
- основной JS-бандл уменьшен примерно с 504 кБ до 381 кБ;
- карта, архив, влияние, живой мир, настройки, бой и подземелья загружаются отдельными файлами;
- предупреждение Vite о чанке больше 500 кБ устранено.

### Контроль симуляции

Добавлены независимые настройки:

- warsEnabled;
- crisesEnabled;
- economicDeclineEnabled;
- borderChangesEnabled;
- cityDestructionEnabled;
- agingEnabled;
- oldAgeDeathEnabled;
- branchSecessionEnabled;
- poachingEnabled;
- maxChronicleEntries;
- compactCardsEnabled;
- devToolsEnabled.

### Время и архивирование

- промотка прекращается при появлении боя, события, подземелья или разбора;
- «до события» использует общий безопасный механизм;
- хроника ограничивается выбранным игроком лимитом;
- старые завершённые экспедиции, последствия, история и снимки карты архивируются;
- активные и пропавшие экспедиции никогда не удаляются очисткой.

### Баланс мира

- безопасность маршрута теперь плавно движется к целевому значению, а не теряется каждый месяц;
- путь может восстановиться после окончания давления;
- экономика городов использует целевые значения продовольствия и процветания;
- демографический рост ограничен вместимостью типа поселения;
- порог превращения города в руины усилен;
- устойчивость государства замедляет развитие кризисов;
- новые кризисы появляются реже.

### Старение

- старение можно полностью отключить;
- естественная смерть управляется отдельным переключателем;
- вероятность смерти зависит от народа и возраста;
- отключение старения сохраняет возраст всех персонажей.

### Dev-панель

- ресурсы;
- полное открытие карты;
- лечение состава;
- аварийное завершение экспедиций;
- генерация кризиса;
- генерация войны;
- аудит на 1/10/50/100 лет;
- экспорт технического JSON-журнала.

## Проверки

- `npm ci`;
- `npm run build`;
- production code splitting;
- двухлетняя симуляция со всеми тяжёлыми системами отключёнными;
- 10/50/100-летние аудиты четырёх пресетов;
- отсутствие массового разрушения городов и торговых путей;
- ограничение хроники на 1200 записей;
- сохранение возраста при отключённом старении;
- сохранение формата `save v7`.

## Результаты аудита

- классический мир: +70% населения за 100 лет, 0 уничтоженных городов, 0 заброшенных маршрутов;
- павшие империи: −26% населения за 100 лет, без полного коллапса;
- дикий фронтир: +41% населения за 100 лет;
- эпоха войны: +16% населения за 100 лет при трёх активных войнах;
- расчёт 100 лет занимает примерно 2–4 секунды в тестовой среде.

## Ограничения

- аудит выполняется синхронно и на 100 годах может кратко заморозить слабый телефон;
- карта всё ещё рендерит все гексы одним SVG;
- экономика остаётся агрегированной;
- старые уже разрушенные города не восстанавливаются при отключении разрушения.
